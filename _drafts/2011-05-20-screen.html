If you're not using the screen command yet, stop what you're doing. Just stop. You really can't go another day without learning this fantastic tool. Take a few minutes to learn it now, and forever be more productive.

Simply put, screen lets you have a multi-terminal environment within a single session. This in itself isn't revolutionary - anyone who works from a gui can have multiple terminal sessions going after all, and who cares how many sessions there are? It's the extras that really get me: multiple terminals in a single session, sharing terminals with other users (that's right, multiple users on one terminal at a time), terminal persistence across disconnects, cut and paste without a mouse... all this, and it's a real terminal application. No gui involved.

Screen


Got a terminal window open? Start up screen with screen. Looks like nothing happened, right? This is something you can tack onto your .bashrc file if you like, it's that transparent. So here's the deal: anything preceded by Ctrl+A is interpreted as a command for Screen, rather than for the terminal.

First thing, let's do something interesting in our existing terminal.
tail -f /var/log/syslog
 ought to do it.

Watch that code fly by! Now open a new screen session, by hitting Ctrl+A, then C. Do something distinctive in this session, like ls -lG /. Note that screen supports colors. Let's see what's going on back in syslog: Ctrl+A, then P. The P and N keys move to the Previous and Next sessions, which makes sense. You can add as many sessions as you like, and scroll through them this way, or using session numbers with Ctrl+A 2 for instance. To close a session, use Ctrl+A k (for Kill).

Now let's start having fun.
